chromium input system

Chromium

Case 1

如果沒有 addEventListener去聽touch event等

.1 touch event進來

.2 因為沒有handler所以直接處理touch event ACK

.3 ACK裡把存下來的gesture送進renderer process的compositor thread

.4 如果compositor thread沒處理送進renderer process的CrRenderMain thread

.5 處理gesture ACK

Case 2

有聽touchevent的話

.1 parse 網頁時has handler被設起來

.2 touch event進來

.3 trigger timer

.4 touch event送到renderer process的compositor thread

.5 如果沒有處理就送到 CrRenderMain thread丟給V8處理

.6 touch event ACK

.7 取gesture

如果沒有preventDefault

.8 gesture送進 renderer process的compositor thread

.9 如果沒有處理再送進CrRenderMain thread

.10 gesture ACK

如果有preventDefault

.8 不send gesture
